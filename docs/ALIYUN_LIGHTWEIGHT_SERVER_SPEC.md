# 阿里云轻量应用服务器规格建议

针对当前博客项目（Next.js 14 + Prisma + PostgreSQL）的部署需求。

---

## 一、项目资源需求概览

| 项目 | 说明 |
|------|------|
| 运行时 | Node.js 20.x、Next.js 14（SSR + API Routes） |
| 数据库 | PostgreSQL（Prisma），可同机或外接 RDS |
| 内存 | `next start` 常驻约 300–800MB，build 时 1.5–2GB+ |
| 磁盘 | 应用 + node_modules + 构建产物约 1–2GB，建议预留 20GB+ |

---

## 二、两种部署方式与规格建议

### 方式一：应用与数据库分离（推荐）

- **轻量服务器**：只跑 Next.js（`next start`）
- **数据库**：阿里云 RDS PostgreSQL 或 云数据库 PolarDB（按量/包月）

| 场景 | 推荐规格 | 说明 |
|------|----------|------|
| 个人/小流量博客 | **2 核 2GB**，系统盘 40–60GB，带宽 3–5M | 足够稳跑 `next start`，build 建议在本地或 CI 完成后再上传 |
| 希望同机做 build | **2 核 4GB**，系统盘 60GB，带宽 3–5M | 可在服务器上执行 `pnpm install && pnpm build`，再 `pnpm start` |
| 有管理后台、多用户 | **2 核 4GB**，带宽 5M | 留余量给并发与后台操作 |

**结论**：优先选 **2 核 2GB**（只跑应用且不在服务器 build）或 **2 核 4GB**（在服务器 build 或想更稳）。

---

### 方式二：应用 + PostgreSQL 同机部署

- 同一台轻量服务器上跑 Node + PostgreSQL

| 场景 | 推荐规格 | 说明 |
|------|----------|------|
| 最低可用 | **2 核 4GB**，系统盘 60GB | Node ~0.5–1GB + PostgreSQL ~0.5–1GB，余量紧张 |
| 推荐 | **2 核 4GB + 数据盘 40–60GB** | 系统盘装系统+应用，数据盘专门放 PostgreSQL 数据与备份，性价比高 |
| 更宽裕 | **2 核 8GB**，系统盘 60GB，可选数据盘 40GB | 建库、迁移、备份更从容，长期更稳 |

**结论**：同机跑数据库时，不建议低于 **2 核 4GB**。选 **2 核 4GB + 数据盘大一点**（如 40–60GB）即可：内存够用，数据盘装数据库和备份，系统盘只跑系统和博客应用，是常见且可行的方案。

---

## 三、规格对照小结

| 部署方式 | 最低建议 | 更稳妥选择 |
|----------|----------|------------|
| 仅应用（数据库用 RDS/外部） | 2 核 2GB，40GB 盘，3M 带宽 | 2 核 4GB，60GB 盘，5M 带宽 |
| 应用 + 数据库同机 | 2 核 4GB，60GB 盘 | 2 核 8GB，60GB 盘 + 数据盘 |

---

## 四、其他建议

1. **系统**：选 **Ubuntu 22.04 LTS** 或 **Alibaba Cloud Linux 3**，便于装 Node 20 与 PostgreSQL。
2. **Build**：若选 2 核 2GB 且不在服务器 build，可在本机或 GitHub Actions 执行 `pnpm build`，把 `.next` 与依赖一起上传，服务器只执行 `pnpm start`（或配合 PM2）。
3. **进程守护**：用 **PM2** 或 **systemd** 跑 `next start`，并做开机自启与重启策略。
4. **反向代理**：用 **Nginx** 做 80/443、SSL 和反向代理到 Node 端口（如 3000）。
5. **数据库**：若用阿里云 RDS，轻量只需 2 核 2GB/4GB；若同机自建 PostgreSQL，务必做好定期备份与安全配置。
6. **数据盘用法**（2 核 4G + 数据盘同机部署时）：数据盘挂载到例如 `/data`，将 PostgreSQL 的数据目录和备份放在数据盘上，系统盘只装系统与 Next.js；这样重装系统时数据不丢，且 I/O 与系统盘分离更稳。

---

## 五、应用镜像选「宝塔」时

若购买轻量时**应用镜像选的是宝塔**，相当于系统已预装宝塔面板，适合同机部署博客 + PostgreSQL，用图形界面管理会省事很多。

### 拿到服务器后建议步骤

1. **登录宝塔**：用阿里云控制台给的公网 IP，在浏览器访问 `http://你的IP:8888`，按提示安装并绑定宝塔账号，改掉默认端口/密码。
2. **在宝塔里安装**  
   - **Nginx**：软件商店安装，用于反向代理和 HTTPS。  
   - **Node.js**：软件商店安装「Node 版本管理」或「PM2」，安装 Node 20.x。  
   - **PostgreSQL**：软件商店安装 PostgreSQL（选当前稳定版），建库、记下用户名密码和端口（默认 5432）。
3. **部署博客项目**  
   - 用宝塔「文件」把项目代码上传到例如 `/www/wwwroot/blog`（或本机 build 好再上传 `.next` + `package.json` 等）。  
   - 在宝塔里用 PM2 添加项目：启动文件选 `node_modules/next/dist/bin/next`，参数 `start`，工作目录选项目根目录；或命令行在项目目录执行 `pnpm install --prod && pnpm build && pnpm start` 后用 PM2 守护。
4. **环境变量**：在 PM2 或系统里配置 `PRISMA_DATABASE_URL`，指向本机 PostgreSQL（如 `postgresql://用户名:密码@127.0.0.1:5432/数据库名`）。
5. **Nginx 反向代理**：在宝塔里为网站添加反向代理，域名或 IP 的 80/443 指向 `http://127.0.0.1:3000`（Next 默认端口），并可选在宝塔里申请 SSL 证书。
6. **数据盘**：若有数据盘，先在宝塔或命令行挂载到 `/data`，再在安装/配置 PostgreSQL 时把数据目录设到数据盘上（或安装后迁移数据目录），便于备份和重装系统不丢数据。

### 注意

- 宝塔应用镜像底层一般是 CentOS/Ubuntu 等，自带 Nginx/MySQL 等可选；你这边要用 **PostgreSQL**，需在软件商店单独装 PostgreSQL，不要和 MySQL 混用。
- 防火墙：在宝塔「安全」里放行 80、443、8888（宝塔面板），必要时放行 5432 仅本机访问即可。

按上述规格即可在阿里云轻量上稳定运行当前博客项目；若后续流量或管理后台使用明显增加，再升级到 2 核 4GB/8GB 或更高即可。
